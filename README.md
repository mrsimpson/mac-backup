# mac-backup

A minimal macOS backup tool for developer machines. Replaces Time Machine for the things that actually matter.

## Why not Time Machine?

Time Machine is great — until you're a developer.

**Developer machines are full of replicated data.** A typical setup has hundreds of git repos (each a full clone), SharePoint/OneDrive synced folders, npm/yarn/cargo caches, Docker layers, and build artifacts. Time Machine backs all of it up faithfully, creating multi-terabyte backup sets for data that either already lives in the cloud or can be regenerated in minutes.

**Time Machine is opaque and often fails on large disks.** When your disk is 1 TB and 600 GB is `node_modules`, Time Machine either takes hours or silently gives up. You rarely notice until you actually need to restore — and then the backup turns out to be weeks old or incomplete.

**Homebrew already is the package manager.** Every tool you installed is declared somewhere. You don't need byte-for-byte backups of binaries that brew can reinstall in minutes. What you need is the *list*.

## What this tool backs up instead

| Category | What | Size (real example) |
|---|---|---|
| **Homebrew** | `Brewfile` — every formula, cask, and tap | **12 KB** (294 packages) |
| **Git repos** | `meta.json` per repo (remotes, branch) + `changes.patch` for dirty worktrees + `local-commits.bundle` for unpushed branches | **81 MB** for 294 repos |
| **Dotfiles** | rsync of selected `~/.*` entries (`.zshrc`, `.ssh`, `.config`, `.kube`, etc.) | **165 MB** (from 275 MB source) |
| **Total** | Everything above | **246 MB** |

Compare that to a Time Machine backup of the same machine: tens of gigabytes at minimum, hundreds more realistically.

The key insight for git: **code is not data**. Every repo has a remote. What's irreplaceable is the diff — uncommitted changes and local branches that haven't been pushed yet. This tool saves exactly that and nothing more.

## Usage

```sh
# First-time setup
./mac-backup config

# Run a backup
./mac-backup backup

# Restore on a new machine
./mac-backup restore
```

### `config`

Interactive wizard that writes `~/.backup-config`:

- Backup destination (default: OneDrive)
- Which dotfiles to include (searchable multi-select picker)
- Git root directory (scanned recursively for all repos)

### `backup`

Runs three steps sequentially, each with a spinner:

1. **Homebrew** — `brew bundle dump` → `backup/homebrew/Brewfile`
2. **Git repos** — scans `GIT_ROOT`, writes one folder per repo with `meta.json` + optional `changes.patch` / `local-commits.bundle`. Shows each repo as it completes:
   ```
   ◆  deutschebahn/my-service [dirty]
   ◆  privat/side-project [unpushed]
   ◆  open-source/tool
   ◇  Git repos: 294 backed up (132 dirty, 89 with unpushed commits) ✓
   ```
3. **Dotfiles** — rsync of selected paths to `backup/dotfiles/`

Ctrl+C stops immediately. Individual step failures are logged but don't abort the rest.

### `restore`

Shows what's available in the backup destination, asks for confirmation, then restores in this order:

1. **Homebrew** — `brew bundle install` from the saved Brewfile
2. **Dotfiles** — rsync back to `~/`, then runs `ssh-add` on all valid OpenSSH private keys using the macOS Keychain for passphrases
3. **Git repos** — clones from origin, re-imports local bundles, applies dirty-state patch

Dotfiles are restored before git repos so SSH keys are available when cloning private repos. Non-OpenSSH files (`.pem`, sockets, `known_hosts`) are skipped. If a key needs a passphrase not in the Keychain, restore stops with instructions to add it manually.

For private HTTPS repos, configure a credential helper before running restore:

```sh
git config --global credential.helper osxkeychain
```

## What it doesn't back up

- **Code** — it's in git remotes already
- **`node_modules`, `dist`, `build`, `.venv`** — regenerated by package managers
- **`~/Library/Application Support`** — app state, caches, preferences that are app-managed
- **Docker images, build caches** — repullable or rebuildable

## Requirements

- macOS, Apple Silicon or Intel
- Node.js 18+
- Homebrew (for the brew step)
- `rsync` (ships with macOS)
- `git` (ships with macOS)

## Install

```sh
git clone <this-repo>
cd mac-backup
npm install
./mac-backup config
```

No global install needed. Run directly from the repo.
